1. Requisitos del Sistema


1.1 Entorno de Producción (Railway)

  • Cuenta en Railway (https://railway.app)
  • Cuenta en GitHub con el repositorio del proyecto
  • Docker instalado (para builds locales opcionales)

    • 1.2 Entorno de Desarrollo Local

  • Python 3.9 o superior
  • Git
  • Git Bash (Windows) o terminal (Linux/macOS)
  • Navegador web moderno


    • 2. Estructura del Proyecto


    sistema_gestion_documental/ ├── Dockerfile # Configuración de contenedor Docker ├── requirements.txt # Dependencias Python ├── run.py # Punto de entrada de la aplicación ├── migrations/ # Migraciones de base de datos (Alembic) │ ├── env.py │ ├── alembic.ini │ └── versions/ │ └── *.py # Archivos de migración └── mi_sistema/ ├── __init__.py # Factory de la aplicación Flask ├── config.py # Configuración por entorno ├── models.py # Modelos SQLAlchemy (31 tablas) ├── routes.py # Rutas y lógica de negocio ├── static/ │ ├── estilos.css # Estilos personalizados │ ├── logo_gobierno.png │ ├── logo_archivos.png │ ├── logo_secretaria.png │ └── *.png # Iconos del menú └── templates/ ├── base.html # Template base con header y footer └── main/ ├── login.html ├── register.html ├── menu.html ├── catalogos.html ├── cat_*.html # Templates de catálogos ├── editar_*.html # Templates de edición ├── cap_expedientes.html ├── auditoria_archivo.html ├── cgca.html ├── ver_cdd.html ├── inventario.html ├── guia_archivos.html └── ficha_valoracion.html


    3. Instalación en Entorno Local


    3.1 Clonar el repositorio

    git clone https://github.com/miklosleingel-png/sistema_gestion_documental.git cd sistema_gestion_documental

    3.2 Crear entorno virtual

    python -m venv venv source venv/Scripts/activate # Windows (Git Bash) source venv/bin/activate # Linux / macOS

    3.3 Instalar dependencias

    pip install -r requirements.txt

    3.4 Configurar variables de entorno

    Crear archivo .env en la raíz del proyecto:

    SECRET_KEY=tu-clave-secreta DATABASE_URL=postgresql://usuario:password@host:puerto/basedatos

    Si no se define DATABASE_URL, el sistema usa SQLite automáticamente para desarrollo local:

    instance/sistema.db

    3.5 Ejecutar migraciones

    export FLASK_APP=run.py flask db upgrade

    3.6 Iniciar el servidor de desarrollo

    python run.py

    Acceder en: http://localhost:8080



    4. Despliegue en Railway


    4.1 Primer despliegue

  • Crear cuenta en https://railway.app
  • Crear nuevo proyecto → GitHub Repository
  • Seleccionar el repositorio `sistema_gestion_documental`
  • Railway detecta el `Dockerfile` y construye automáticamente
  • Agregar servicio PostgreSQL desde el dashboard
  • Configurar variables de entorno en el servicio:

    • DATABASE_URL=postgresql://postgres:PASSWORD@postgres.railway.internal:5432/railway SECRET_KEY=tu-clave-secreta

  • Railway asigna una URL pública automáticamente

    • 4.2 Migraciones en producción

    Ejecutar desde el entorno local con la URL pública de PostgreSQL:

    export DATABASE_URL="postgresql://postgres:PASSWORD@metro.proxy.rlwy.net:PUERTO/railway" export FLASK_APP=run.py flask db upgrade

    4.3 Actualizaciones

    El despliegue es automático:

    git add . git commit -m "descripción del cambio" git push

    Railway detecta el push y redespliega en aproximadamente 2-3 minutos.



    5. Dockerfile


    FROM python:3.9-slim WORKDIR /app RUN apt-get update && apt-get install -y \ libpq-dev gcc && rm -rf /var/lib/apt/lists/* COPY requirements.txt . RUN pip install --no-cache-dir -r requirements.txt COPY . . EXPOSE 8080 CMD exec gunicorn --bind 0.0.0.0:8080 --workers 1 --threads 8 --timeout 0 run:app

    7. Configuración de la Aplicación


    7.1 __init__.py — Factory de Flask

    La función create_app() inicializa la aplicación con:

  • Configuración de base de datos (PostgreSQL en producción, SQLite en desarrollo)
  • Registro de extensiones: SQLAlchemy, LoginManager, Migrate
  • Registro del Blueprint principal main
  • Context processor para la función now() (fecha automática en templates)

    • 7.2 config.py — Entornos

    class DevelopmentConfig: # FLASK_ENV=development → debug=True class ProductionConfig: # FLASK_ENV=production → debug=False class TestingConfig: # FLASK_ENV=testing → testing=True


    8. Base de Datos


    8.1 Esquema simplificado


    cat_cgca ──────────────── cat_clave_cgca (maestra) │ ┌─────────────────┼─────────────────────┐ │ │ │ clavecgca_funcion clavecgca_fundamento clavecgca_procedimiento │ │ │ cat_funcion cat_leyes cat_procedimientos │ cat_actividades (id_ua / id_area_ext)

    8.2 Crear migraciones nuevas

    Cuando se modifica models.py:

    flask db migrate -m "descripción del cambio" flask db upgrade git add migrations/ git commit -m "nueva migración" git push

    8.3 Rollback de migración

    flask db downgrade


    9. Gestión de Usuarios


    9.1 Crear primer usuario administrador

    Acceder a /register en el navegador (solo disponible si no hay usuarios registrados, o con permisos de admin).


    9.2 Roles disponibles

    Deben existir en la tabla roles antes de registrar usuarios:

    INSERT INTO roles (id, nombre, descripcion) VALUES (1, 'Administrador', 'Acceso total al sistema'), (2, 'Capturista', 'Captura de expedientes'), (3, 'Consulta', 'Solo lectura');


    10. Variables de Entorno


    VariableDescripciónRequerida

    11. Archivos a Ignorar en Git


    El archivo .gitignore debe incluir:

    venv/ __pycache__/ *.pyc instance/ *.db .env cloud-sql-proxy migrations/versions/ *.gnumeric


    12. Solución de Problemas Comunes


    ProblemaCausa probableSolución
    Error 500 en producciónTemplate corrupto o error en rutaRevisar Deploy Logs en Railway
    `no such table`Base de datos sin migrarEjecutar `flask db upgrade`
    `could not translate host name`DATABASE_URL usa host interno fuera de RailwayCambiar a URL pública (`metro.proxy.rlwy.net`)
    `$PORT is not a valid port number`Procfile o Dockerfile con variable sin expandirUsar puerto fijo `0.0.0.0:8080` en Dockerfile
    Login falla con SQLiteDATABASE_URL no configuradaAgregar variable en Railway → Variables
    Template con 5000+ líneasArchivo corrupto por escritura incorrectaRestaurar desde outputs del proyecto
